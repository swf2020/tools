# Java注解处理器(APT)编译期代码生成技术文档

## 1. 概述

### 1.1 什么是注解处理器(APT)
Java注解处理器(Annotation Processing Tool)是Java编译器(javac)的一部分，用于在编译阶段处理源代码中的注解，并生成额外的源代码文件。APT基于JSR 269标准实现，提供了一种元编程机制，使得开发者可以在编译时进行代码分析、验证和生成。

### 1.2 核心优势
- **编译时执行**：在编译阶段运行，不影响运行时性能
- **类型安全**：生成的代码经过完整编译检查
- **零运行时依赖**：生成的代码是普通Java类，无需额外运行时库
- **IDE友好**：主流IDE（IntelliJ IDEA, Eclipse）都支持APT集成

## 2. 工作原理架构

### 2.1 处理流程
```
源代码 → 词法/语法分析 → 抽象语法树(AST) → 注解处理器 → 生成的源代码 → 再次编译
```

### 2.2 核心组件
```java
// 处理循环：编译器多次调用处理器，直到没有新文件生成
Round 1: 处理原始源代码中的注解，生成新文件
Round 2: 处理新生成文件中的注解
...
Round N: 直到没有新文件生成
```

## 4. 实现注解处理器

### 4.1 定义注解
```java
@Target(ElementType.TYPE)  // 可用于类、接口、枚举
@Retention(RetentionPolicy.SOURCE) // 编译时保留
public @interface Builder {
    String name() default "";
    boolean fluent() default true;
}
```

### 4.2 实现处理器
```java
@SupportedAnnotationTypes("com.example.Builder")
@SupportedSourceVersion(SourceVersion.RELEASE_11)
public class BuilderProcessor extends AbstractProcessor {
    
    private Elements elementUtils;
    private Types typeUtils;
    private Filer filer;
    private Messager messager;
    
    @Override
    public synchronized void init(ProcessingEnvironment processingEnv) {
        super.init(processingEnv);
        this.elementUtils = processingEnv.getElementUtils();
        this.typeUtils = processingEnv.getTypeUtils();
        this.filer = processingEnv.getFiler();
        this.messager = processingEnv.getMessager();
    }
    
    @Override
    public boolean process(Set<? extends TypeElement> annotations, 
                          RoundEnvironment roundEnv) {
        for (TypeElement annotation : annotations) {
            Set<? extends Element> annotatedElements = 
                roundEnv.getElementsAnnotatedWith(annotation);
            
            for (Element element : annotatedElements) {
                if (element.getKind() != ElementKind.CLASS) {
                    error(element, "Only classes can be annotated with @Builder");
                    return true;
                }
                
                TypeElement typeElement = (TypeElement) element;
                try {
                    generateBuilder(typeElement);
                } catch (IOException e) {
                    error(element, "Failed to generate builder: " + e.getMessage());
                }
            }
        }
        return true;
    }
    
    private void generateBuilder(TypeElement typeElement) throws IOException {
        String className = typeElement.getSimpleName().toString();
        String packageName = elementUtils.getPackageOf(typeElement).toString();
        String builderClassName = className + "Builder";
        
        // 收集字段信息
        List<VariableElement> fields = new ArrayList<>();
        for (Element enclosed : typeElement.getEnclosedElements()) {
            if (enclosed.getKind() == ElementKind.FIELD) {
                fields.add((VariableElement) enclosed);
            }
        }
        
        // 生成源代码
        JavaFileObject builderFile = filer.createSourceFile(
            packageName + "." + builderClassName, typeElement);
        
        try (PrintWriter writer = new PrintWriter(builderFile.openWriter())) {
            // 包声明
            writer.println("package " + packageName + ";");
            writer.println();
            
            // 类注释
            writer.println("/**");
            writer.println(" * Generated by BuilderProcessor");
            writer.println(" */");
            
            // 类声明
            writer.println("public class " + builderClassName + " {");
            writer.println();
            
            // 字段声明
            for (VariableElement field : fields) {
                String fieldName = field.getSimpleName().toString();
                String fieldType = field.asType().toString();
                writer.println("    private " + fieldType + " " + fieldName + ";");
            }
            writer.println();
            
            // setter方法（链式调用）
            for (VariableElement field : fields) {
                String fieldName = field.getSimpleName().toString();
                String fieldType = field.asType().toString();
                String methodName = "set" + 
                    Character.toUpperCase(fieldName.charAt(0)) + 
                    fieldName.substring(1);
                
                writer.println("    public " + builderClassName + " " + 
                    methodName + "(" + fieldType + " " + fieldName + ") {");
                writer.println("        this." + fieldName + " = " + fieldName + ";");
                writer.println("        return this;");
                writer.println("    }");
                writer.println();
            }
            
            // build方法
            writer.println("    public " + className + " build() {");
            writer.println("        " + className + " instance = new " + className + "();");
            for (VariableElement field : fields) {
                String fieldName = field.getSimpleName().toString();
                writer.println("        instance." + fieldName + " = this." + fieldName + ";");
            }
            writer.println("        return instance;");
            writer.println("    }");
            
            writer.println("}");
        }
    }
    
    private void error(Element element, String message) {
        messager.printMessage(Diagnostic.Kind.ERROR, message, element);
    }
}
```

### 6.3 使用JavaPoet简化代码生成
```java
// 添加依赖
// implementation 'com.squareup:javapoet:1.13.0'

private void generateWithJavaPoet(TypeElement typeElement) throws IOException {
    String className = typeElement.getSimpleName().toString();
    String builderClassName = className + "Builder";
    
    TypeSpec.Builder builder = TypeSpec.classBuilder(builderClassName)
        .addModifiers(Modifier.PUBLIC)
        .addJavadoc("Generated by BuilderProcessor\n");
    
    // 添加字段
    for (Element element : typeElement.getEnclosedElements()) {
        if (element.getKind() == ElementKind.FIELD) {
            VariableElement field = (VariableElement) element;
            String fieldName = field.getSimpleName().toString();
            TypeName fieldType = TypeName.get(field.asType());
            
            // 添加字段
            builder.addField(fieldType, fieldName, Modifier.PRIVATE);
            
            // 添加setter方法
            MethodSpec setter = MethodSpec.methodBuilder("set" + 
                Character.toUpperCase(fieldName.charAt(0)) + fieldName.substring(1))
                .addModifiers(Modifier.PUBLIC)
                .returns(ClassName.get("", builderClassName))
                .addParameter(fieldType, fieldName)
                .addStatement("this.$N = $N", fieldName, fieldName)
                .addStatement("return this")
                .build();
            builder.addMethod(setter);
        }
    }
    
    // 添加build方法
    MethodSpec buildMethod = MethodSpec.methodBuilder("build")
        .addModifiers(Modifier.PUBLIC)
        .returns(ClassName.get("", className))
        .addStatement("$T instance = new $T()", 
            ClassName.get("", className), ClassName.get("", className))
        .addStatement("return instance")
        .build();
    builder.addMethod(buildMethod);
    
    // 生成文件
    JavaFile javaFile = JavaFile.builder(
        elementUtils.getPackageOf(typeElement).toString(),
        builder.build()
    ).build();
    
    javaFile.writeTo(filer);
}
```

## 10. 常见问题与解决方案

### 10.1 处理器未执行
**可能原因**：
1. 未正确配置META-INF/services
2. 注解保留策略不是SOURCE或CLASS
3. 编译顺序问题

**解决方案**：
```bash
# 清理并重新编译
mvn clean compile

# 查看处理器是否被发现
mvn compile -X
```

### 10.2 生成的代码无法引用
**解决方案**：
```java
// 在生成代码时指定originatingElements
JavaFileObject file = filer.createSourceFile(
    generatedClassName, 
    originatingElement // 原始注解所在的元素
);
```

### 10.3 处理循环依赖
```java
@Override
public boolean process(Set<? extends TypeElement> annotations, 
                      RoundEnvironment roundEnv) {
    if (roundEnv.processingOver()) {
        // 最后一轮处理，进行清理工作
        return false;
    }
    
    if (!roundEnv.errorRaised()) {
        // 正常处理逻辑
    }
    
    return true; // 允许其他处理器继续处理
}
```
