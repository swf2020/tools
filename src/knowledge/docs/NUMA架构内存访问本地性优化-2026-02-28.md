# NUMA架构内存访问本地性优化技术文档

## 1. NUMA架构概述

### 1.1 NUMA基本概念
NUMA（Non-Uniform Memory Access，非一致内存访问）是一种多处理器计算机内存设计架构，其中内存访问时间取决于内存位置相对于处理器的位置。在NUMA系统中，每个处理器拥有本地内存，访问速度快；而访问其他处理器的远程内存则需要通过互连链路，速度较慢。

### 1.2 NUMA系统组成
- **节点（Node）**：包含处理器核心、内存控制器和本地内存
- **本地内存（Local Memory）**：与节点直接相连的内存
- **远程内存（Remote Memory）**：属于其他节点的内存
- **互连链路（Interconnect）**：连接各节点的通信通道

## 2. NUMA内存访问性能影响

### 2.1 访问延迟差异
- 本地内存访问：延迟低，带宽高
- 远程内存访问：延迟通常增加1.5-3倍，带宽降低

### 2.2 性能影响因素
```
本地访问 vs 远程访问性能对比：
1. 延迟：本地 < 远程
2. 带宽：本地 > 远程
3. 互连拥塞：远程访问可能导致链路拥塞
```

## 3. NUMA感知的应用程序优化策略

### 3.1 数据初始化与分配策略

#### 3.1.1 首次接触策略（First-Touch）
```c
// 正确：数据在使用的线程上初始化
#pragma omp parallel for
for (int i = 0; i < N; i++) {
    data[i] = 0.0;  // 初始化在执行的线程本地内存
}

// 错误：主线程初始化所有数据
for (int i = 0; i < N; i++) {
    data[i] = 0.0;  // 所有数据分配在主线程本地内存
}
```

#### 3.1.2 显式内存绑定
```bash
# 使用numactl控制内存分配策略
numactl --localalloc ./application      # 仅本地分配
numactl --preferred=0 ./application     # 优先节点0
numactl --interleave=all ./application  # 交错分配
```

#### 3.2.2 NUMA感知的内存分配
```c
// 使用numa_alloc_local分配本地内存
#include <numa.h>

void* allocate_local_memory(size_t size) {
    if (numa_available() < 0) {
        return malloc(size);
    }
    int current_node = numa_preferred();
    void* ptr = numa_alloc_onnode(size, current_node);
    
    // 绑定内存到当前节点
    numa_tonode_memory(ptr, size, current_node);
    return ptr;
}
```

                         (pattern->local_access + pattern->remote_access);
    return remote_ratio > threshold;

    data = new_data;
}

## 6. 最佳实践总结

### 6.1 设计原则
1. **数据本地化**：尽量让数据靠近使用它的计算核心
2. **平衡分配**：避免某个NUMA节点过载
3. **最小化远程访问**：优化数据布局减少跨节点访问
4. **适当复制**：对频繁读取的只读数据可考虑适当复制

### 6.2 实施步骤
1. 分析应用程序的内存访问模式
2. 识别热点数据结构和关键代码路径
3. 实现NUMA感知的内存分配
4. 绑定线程到合适的CPU核心
5. 验证优化效果并迭代改进

### 6.3 常见陷阱
1. 忽略"首次接触"策略的影响
2. 不正确的线程绑定导致资源争用
3. 过度优化导致的代码复杂性增加
4. 忽视硬件特定的NUMA特性

## 7. 示例代码：NUMA优化的矩阵乘法

```c
#include <numa.h>
#include <omp.h>
#include <stdlib.h>

void numa_optimized_matmul(double* A, double* B, double* C, 
                           int n, int block_size) {
    int num_nodes = numa_max_node() + 1;
    
    #pragma omp parallel
    {
        int thread_id = omp_get_thread_num();
        int node_id = thread_id % num_nodes;
        
        // 绑定线程到对应NUMA节点
        numa_run_on_node(node_id);
        
        // 计算分配给当前线程的数据块
        int block_start = (n / omp_get_num_threads()) * thread_id;
        int block_end = (thread_id == omp_get_num_threads() - 1) ? 
                       n : (n / omp_get_num_threads()) * (thread_id + 1);
        
        // 处理本地数据块
        for (int i = block_start; i < block_end; i += block_size) {
            for (int j = 0; j < n; j += block_size) {
                for (int k = 0; k < n; k += block_size) {
                    // 块矩阵乘法
                    for (int ii = i; ii < min(i+block_size, block_end); ii++) {
                        for (int kk = k; kk < min(k+block_size, n); kk++) {
                            for (int jj = j; jj < min(j+block_size, n); jj++) {
                                C[ii*n + jj] += A[ii*n + kk] * B[kk*n + jj];
                            }
                        }
                    }
                }
            }
        }
    }
}
```
