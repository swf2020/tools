# 分布式限流技术文档：基于Redis+Lua的原子计数方案

## 1. 概述

### 1.1 文档目的
本文档详细阐述基于Redis和Lua脚本实现的分布式限流解决方案，该方案通过原子操作确保在分布式环境下的高并发限流准确性。

### 1.2 适用场景
- 微服务架构下的API限流
- 秒杀、抢购等高并发场景
- 防止恶意请求和爬虫攻击
- 服务资源保护与负载均衡

## 2. 技术背景

### 2.1 分布式限流挑战
在分布式系统中，传统单机限流方案（如Guava RateLimiter）存在以下问题：
- 限流状态无法跨节点共享
- 集群整体流量难以精确控制
- 存在竞态条件导致限流失效

### 2.2 为什么选择Redis+Lua
- **Redis特性**：单线程模型、高性能、支持原子操作
- **Lua脚本优势**：在Redis中原子性执行，避免多命令竞态条件
- **数据一致性**：保证分布式环境下限流计数的准确性

## 3. 核心设计

### 3.1 限流算法选择
采用**固定窗口计数器算法**的增强版本，结合滑动窗口思想提高精度。

### 3.2 系统架构
```
客户端请求 → 网关/中间件 → Redis限流服务 → 业务服务
                     ↓
                 限流决策
                     ↓
              通过/拒绝/排队
```

## 4. 关键技术实现

### 4.1 Redis数据结构设计

```lua
-- 值：当前窗口请求计数
```

### 4.2 Lua脚本实现原子操作
```lua
-- KEYS[1]: 限流资源标识
-- ARGV[1]: 时间窗口大小（秒）
-- ARGV[2]: 最大请求阈值
-- ARGV[3]: 当前时间戳

local key = KEYS[1]
local window = tonumber(ARGV[1])
local limit = tonumber(ARGV[2])
local currentTime = tonumber(ARGV[3])

-- 清理过期窗口（滑动窗口）
local clearBefore = currentTime - window
redis.call('ZREMRANGEBYSCORE', key, 0, clearBefore)

-- 获取当前窗口请求数
local currentCount = redis.call('ZCARD', key)

-- 判断是否超过限制
if currentCount >= limit then
    return 0 -- 限流
end

-- 添加当前请求
redis.call('ZADD', key, currentTime, currentTime .. ':' .. math.random())
redis.call('EXPIRE', key, window)

return 1 -- 放行
```

### 4.3 高级版本：令牌桶算法实现
```lua
-- KEYS[1]: 令牌桶key
-- KEYS[2]: 最后刷新时间key
-- ARGV[1]: 令牌容量
-- ARGV[2]: 填充速率（令牌/秒）
-- ARGV[3]: 请求令牌数

local tokensKey = KEYS[1]
local timestampKey = KEYS[2]

local capacity = tonumber(ARGV[1])
local fillRate = tonumber(ARGV[2])
local requested = tonumber(ARGV[3])
local now = tonumber(redis.call('TIME')[1])

-- 获取最后刷新时间和当前令牌数
local lastTimestamp = redis.call('GET', timestampKey)
local currentTokens = tonumber(redis.call('GET', tokensKey) or capacity)

if lastTimestamp then
    local elapsed = now - tonumber(lastTimestamp)
    local filledTokens = math.floor(elapsed * fillRate)
    currentTokens = math.min(capacity, currentTokens + filledTokens)
end

-- 检查令牌是否足够
if currentTokens < requested then
    -- 更新时间和令牌数
    redis.call('SET', timestampKey, now)
    redis.call('SET', tokensKey, currentTokens)
    return 0 -- 令牌不足
end

-- 消耗令牌
currentTokens = currentTokens - requested
redis.call('SET', timestampKey, now)
redis.call('SET', tokensKey, currentTokens)

return 1 -- 成功获取令牌
```

## 5. 集成实现

### 5.1 Spring Boot集成示例
```java
@Component
public class RedisRateLimiter {
    
    private final RedisTemplate<String, Object> redisTemplate;
    private final DefaultRedisScript<Long> rateLimitScript;
    
    public boolean tryAcquire(String resourceKey, int limit, int windowSec) {
        List<String> keys = Collections.singletonList(
            "rate_limit:" + resourceKey
        );
        
        long currentTime = System.currentTimeMillis() / 1000;
        Object result = redisTemplate.execute(
            rateLimitScript,
            keys,
            windowSec,
            limit,
            currentTime
        );
        
        return Long.valueOf(1).equals(result);
    }
    
    // 初始化Lua脚本
    @PostConstruct
    public void init() {
        rateLimitScript = new DefaultRedisScript<>();
        rateLimitScript.setScriptSource(new ResourceScriptSource(
            new ClassPathResource("scripts/rate_limiter.lua")
        ));
        rateLimitScript.setResultType(Long.class);
    }
}
```

### 5.2 网关层集成（Spring Cloud Gateway）
```yaml
# application.yml
spring:
  cloud:
    gateway:

          uri: lb://user-service
```
