# 布隆过滤器误判率计算：哈希函数数量与位数组大小的数学关系

## 1. 核心公式与变量定义

### 1.1 主要数学关系

布隆过滤器的误判率（false positive rate）可通过以下公式计算：

```
p ≈ (1 - e^(-k * n / m))^k
```

**变量说明：**
- **p**：误判率（0 ≤ p ≤ 1）
- **k**：哈希函数的数量
- **n**：预期插入的元素数量
- **m**：位数组的大小（位数）
- **e**：自然对数的底数（约等于2.71828）

## 2. 参数对误判率的影响分析

### 2.1 位数组大小 (m) 的影响

**固定条件**：k=3, n=100,000

| 位数组大小 (m) | 误判率 (p) | 存储空间 (MB) |
|----------------|------------|---------------|
| 479,253        | ≈ 1.00%    | 0.057 MB      |
| 958,506        | ≈ 0.10%    | 0.114 MB      |
| 1,437,759      | ≈ 0.01%    | 0.171 MB      |
| 2,396,265      | ≈ 0.001%   | 0.286 MB      |

**结论**：位数组大小与误判率呈指数负相关关系，m越大，误判率越低。

### 2.2 哈希函数数量 (k) 的影响

**固定条件**：m=1,000,000, n=100,000

| 哈希函数数量 (k) | 误判率 (p) | 计算开销 |
|------------------|------------|----------|
| 1                | ≈ 9.52%    | 低       |
| 3                | ≈ 2.13%    | 中等     |
| 5                | ≈ 1.10%    | 较高     |
| 7                | ≈ 1.55%    | 高       |
| 10               | ≈ 4.73%    | 很高     |

**重要发现**：存在一个最优的k值，使误判率最低（当k ≈ (m/n) * ln2时）

## 3. 最优参数推导

### 3.1 最优哈希函数数量公式

在给定m和n的情况下，最小化误判率的最优k值为：

```
k_optimal = (m / n) * ln(2) ≈ 0.693 * (m / n)
```

### 3.2 给定误判率确定位数组大小

如果预设目标误判率为p，所需的最小位数为：

```
m_min = - (n * ln(p)) / (ln(2)^2)
```

## 4. 计算示例

### 4.1 示例场景

假设需要设计一个布隆过滤器，要求：
- 存储元素数量：n = 1,000,000
- 可接受误判率：p ≤ 0.1%

**步骤1：计算所需位数组大小**
```
m = - (1,000,000 * ln(0.001)) / (ln(2)^2)
  ≈ - (1,000,000 * -6.9078) / (0.4805)
  ≈ 14,377,590 bits
  ≈ 1.71 MB
```

**步骤2：计算最优哈希函数数量**
```
k_optimal = (14,377,590 / 1,000,000) * ln(2)
          ≈ 14.3776 * 0.6931
          ≈ 9.96 ≈ 10
```

**步骤3：验证实际误判率**
```
p_actual = (1 - e^(-10 * 1,000,000 / 14,377,590))^10
         ≈ (1 - e^(-0.6956))^10
         ≈ (1 - 0.499)^10
         ≈ 0.001 ≈ 0.1%
```

## 附录：常见问题解答

**Q：增加哈希函数一定能降低误判率吗？**
A：不一定。当k过大时，位数组过早饱和，反而增加误判率。

**Q：如何选择哈希函数的具体实现？**
A：推荐使用MurmurHash、CityHash等非加密哈希，平衡速度与分布均匀性。

**Q：实际误判率与理论计算有差异怎么办？**
A：可能原因包括哈希函数碰撞、元素分布不均匀等，建议增加10-20%的安全余量。
