# Elasticsearch 相关性评分：BM25 算法详解

**文档版本**：1.0
**发布日期**：2023-10-27
**适用版本**：Elasticsearch 5.0+

---

## 1. 摘要

本文档旨在深入解析 Elasticsearch 中默认使用的相关性评分算法 **BM25 (Best Matching 25)**。内容涵盖其设计哲学、数学模型、核心参数，以及在 Elasticsearch 中的配置与应用实践。本文假设读者已具备 Elasticsearch 基础查询知识。

## 2. 引言：为什么需要相关性评分？

在全文搜索引擎中，当用户提交一个查询（如 “机器学习算法”）时，系统通常会返回成千上万个包含关键词的文档。**相关性评分（Relevance Scoring）** 的核心任务，就是根据查询条件，为每个匹配的文档计算一个数值分数（`_score`），并据此对结果进行降序排列，将最符合用户意图的文档置于顶部。

Elasticsearch 在 5.x 版本之后，将默认的评分算法从经典的 **TF-IDF** 切换为更先进的 **BM25**。这一变更是为了处理现代搜索应用中遇到的一些关键问题，如对文档长度更合理的归一化、词频饱和度的优化等。

## 3. 从 TF-IDF 到 BM25

### 3.1 TF-IDF 的局限性
传统 TF-IDF 是 BM25 的基石，其评分公式核心由两部分组成：
*   **TF (Term Frequency)**： 词频，一个词在文档中出现的次数越多，相关性越高。
*   **IDF (Inverse Document Frequency)**： 逆文档频率，一个词在所有文档中出现的频率越低（即越罕见），其区分度越高，权重越大。

TF-IDF 的简单组合存在明显缺陷：
1.  **TF 线性增长问题**： 词频对评分的影响是线性的，一个词出现 20 次并不一定比出现 10 次的相关性高一倍，可能导致内容堆砌关键词的文档获得不合理的高分。
2.  **文档长度处理粗糙**： 虽然考虑了对长文档的惩罚（通常通过字段长度归一化），但方式较为简单，对不同场景的适应性不足。

### 3.2 BM25 的改进
BM25 源自概率检索模型，是一种 **Bag-of-Words** 模型。它并非一个单一公式，而是一个评分函数的家族。其核心思想在保留 TF-IDF 框架精髓的同时，通过**非线性词频饱和度**和**更灵活的文档长度归一化**来解决上述问题，从而在实际应用中（特别是长文本、网页搜索）表现更优。

## 4. BM25 算法核心解析

一个典型的 BM25 评分公式可以表示为对查询中每个词项的贡献进行求和：

**Score(D, Q) = Σ [ IDF(qᵢ) * ( TF(qᵢ, D) * (k₁ + 1) ) / ( TF(qᵢ, D) + k₁ * (1 - b + b * (|D| / avgdl)) ) ]**

其中：
*   **D**: 当前文档
*   **Q**: 查询，由多个词项 `qᵢ` 组成
*   **Σ**: 对查询中所有词项求和

让我们分解其中的关键组件：

### 4.1 IDF (逆文档频率)
与 TF-IDF 中的 IDF 概念相似，衡量词项的“独家性”。在 Elasticsearch 的 Lucene 实现中，其计算通常为：
`IDF(qᵢ) = log( 1 + ( (N - n(qᵢ) + 0.5) / (n(qᵢ) + 0.5) ) )`
*   **N**: 索引中的文档总数。
*   **n(qᵢ)**: 包含词项 `qᵢ` 的文档数。
*   **作用**： 包含罕见词（`n(qᵢ)` 小）的文档会获得更高的 IDF 权重。

### 4.2 TF (词频) 与饱和度控制 - `k₁` 参数
这是 BM25 与 TF-IDF 最显著的区别。BM25 使用一个**饱和函数**来调节词频的影响：
`TF_BM25 = (tf * (k₁ + 1)) / (tf + k₁)`

*   **tf**: 词项在文档中的原始频率。
*   **k₁ (可调参数，默认 `1.2`)**： 控制词频饱和度的“曲率”。
    *   当 `tf` 远小于 `k₁` 时，`TF_BM25` 近似线性增长。
    *   当 `tf` 持续增大并远超 `k₁` 时，`TF_BM25` 的增长会放缓并无限趋近于上限 `(k₁ + 1)`。
    *   **直观理解**： `k₁` 越小，饱和度来得越快（词频重要性快速饱和）；`k₁` 越大，越接近线性（允许词频有更大影响）。这有效防止了关键词堆砌的文档获得不成比例的高分。

### 4.3 文档长度归一化 - `b` 参数
BM25 巧妙地考虑了文档长度，并将其与 TF 部分整合：
`Norm(D) = 1 - b + b * (|D| / avgdl)`

*   **|D|**: 当前文档的字段长度（词项数）。
*   **avgdl**: 索引中所有文档该字段的平均长度。
*   **b (可调参数，默认 `0.75`)**： 控制长度归一化的强度。
    *   `b = 0`： 完全禁用长度归一化。
    *   `b = 1`： 启用完全的长度归一化。
    *   `0 < b < 1`： 介于两者之间。较长的文档会因其长度受到惩罚（TF 贡献被稀释），较短的文档则会获得增益。
    *   **直观理解**： 对于像“新闻正文”这样的字段，我们可能希望稍长的文章（内容更全面）不处于劣势，可以设置较低的 `b`（如 `0.3`）。对于“标题”或“摘要”字段，可以设置较高的 `b`（如 `0.9`），以确保长度一致性。

## 5. 在 Elasticsearch 中应用与调整 BM25

### 5.1 映射定义
在创建索引映射时，可以为特定字段指定使用 `BM25` 相似度算法，并设置自定义的 `k1` 和 `b` 参数。

```json
PUT /my-index
{
  "mappings": {
    "properties": {
      "title": {
        "type": "text",
        "similarity": "my_bm25" // 使用自定义相似度
      },
      "content": {
        "type": "text" // 使用默认的 BM25 相似度
      }
    }
  },
  "settings": {
    "index": {
      "similarity": {
        "my_bm25": { // 定义自定义相似度
          "type": "BM25",
          "k1": 1.3,
          "b": 0.8
        }
      }
    }
  }
}
```

### 5.2 参数调优建议
*   **默认值 (`k1=1.2, b=0.75`)**： 这是一个基于大量实验的、普遍表现良好的起点。**大多数情况下，无需修改。**
*   **何时调 `k1`**：
    *   如果你希望词频的影响更持久（例如，在专利或学术文献搜索中，一个术语重复出现可能确实至关重要），可以适当**增大 `k1`** (如 1.5-2.0)。
    *   如果发现关键词堆砌的垃圾内容排名靠前，可以尝试**减小 `k1`** (如 0.8-1.0)，让词频更快饱和。
*   **何时调 `b`**：
    *   对于需要**强烈惩罚长文档**的字段（如“标签”、“作者”），设置 **`b` 接近 1**。
    *   对于**长度变化大但不应过度惩罚**的字段（如“产品描述”、“文章正文”），可以**降低 `b`** (如 0.3-0.6)。
*   **调优方法**：
    1.  准备一个具有代表性查询和人工标注相关性（好/中/差）的**测试文档集**。
    2.  使用 **Elasticsearch 的 Rank Evaluation API** 来定量评估不同参数配置下的搜索质量（如 NDCG, Precision@k）。
    3.  进行 A/B 测试，观察用户行为指标（如点击率、停留时间）。

### 5.3 查看评分解释
使用 `explain` API 是理解 BM25 如何为特定文档计算分数的利器。

```json
GET /my-index/_search
{
  "explain": true,
  "query": {
    "match": {
      "content": "elasticsearch bm25"
    }
  }
}
```
在返回结果中，可以清晰地看到 IDF 值、TF 值、字段长度归一化因子以及它们是如何组合成最终 `_score` 的。

## 6. 总结

*   **BM25** 是 Elasticsearch 中现代、高效且默认的相关性评分算法，它改进了传统 TF-IDF 的不足。
*   其核心优势在于**非线性词频饱和度（`k1` 参数）** 和**可配置的文档长度归一化（`b` 参数）**，使得评分更符合人类对相关性的感知。
*   在实践中，应首先依赖其**合理的默认参数**。只有在有明确的业务需求、可衡量的评估指标和测试集的情况下，才建议进行精细化的参数调优。
*   理解 BM25 的原理，结合 `explain` API，能够帮助开发者更好地诊断搜索排名问题，并构建出更符合用户期望的搜索体验。

**附录：BM25 与 TF-IDF 行为对比简表**

| 特性 | TF-IDF | BM25 |
| :--- | :--- | :--- |
| **TF 影响** | 线性增长 | 非线性饱和（受 `k1` 控制） |
| **长度归一化** | 通常较简单（如余弦归一化） | 灵活，与 TF 整合（受 `b` 控制） |
| **可调性** | 参数较少，调优空间有限 | 通过 `k1` 和 `b` 提供明确的调优旋钮 |
| **适用场景** | 经典文本检索，简单场景 | 现代网页、长文档、内容质量不均的通用搜索 |
