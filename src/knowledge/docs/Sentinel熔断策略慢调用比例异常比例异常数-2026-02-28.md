# Sentinel熔断策略技术文档：慢调用比例/异常比例/异常数

## 1. 概述

熔断器是微服务架构中关键的稳定性保障组件，用于防止服务雪崩效应。Sentinel作为阿里开源的流量控制组件，提供了多种熔断策略，其中**慢调用比例**、**异常比例**和**异常数**是最核心的三种熔断策略。本文档详细阐述这三种策略的原理、配置和使用方法。

## 2. 熔断机制基础概念

### 2.1 熔断器状态机
```
CLOSED(关闭) → OPEN(打开) → HALF-OPEN(半开) → CLOSED(关闭)
```
- **CLOSED**：正常状态，所有请求通过
- **OPEN**：熔断状态，拒绝所有请求
- **HALF-OPEN**：试探恢复状态，允许部分请求通过以检测服务是否恢复

### 2.2 核心参数
- **熔断时长**：熔断触发后，进入OPEN状态的持续时间
- **统计窗口**：指标统计的时间窗口长度
- **最小请求数**：触发熔断的最小请求阈值
- **阈值**：触发熔断的临界值（比例或数量）

## 3. 慢调用比例策略(Slow Ratio)

### 3.1 策略原理
基于响应时间的熔断策略。当**慢调用比例**（响应时间超过阈值的请求比例）超过设定阈值，且在统计窗口内的请求数达到最小请求数时触发熔断。

### 3.2 规则配置参数
```java
public class DegradeRule {
    private String resource;          // 资源名称
    private double count;             // 响应时间阈值（单位：ms）
    private int timeWindow;           // 熔断时长（单位：s）
    private int grade = RuleConstant.DEGRADE_GRADE_RT; // 策略类型：慢调用比例
    private double slowRatioThreshold; // 慢调用比例阈值（0.0-1.0）
    private int minRequestAmount;     // 最小触发请求数
    private int statIntervalMs;       // 统计窗口（单位：ms）
}
```

### 3.3 配置示例
```java
List<DegradeRule> rules = new ArrayList<>();
DegradeRule rule = new DegradeRule("orderService")
    .setGrade(RuleConstant.DEGRADE_GRADE_RT)      // 慢调用比例策略
    .setCount(500)                                // RT阈值：500ms
    .setSlowRatioThreshold(0.5)                   // 比例阈值：50%
    .setMinRequestAmount(10)                      // 最小请求数：10
    .setStatIntervalMs(10000)                     // 统计窗口：10秒
    .setTimeWindow(5);                            // 熔断时长：5秒
rules.add(rule);
DegradeRuleManager.loadRules(rules);
```

### 3.4 触发条件
1. 统计窗口内请求数 ≥ minRequestAmount
2. 慢调用比例 ≥ slowRatioThreshold
   ```
   慢调用比例 = (响应时间 > count的请求数) / 总请求数
   ```

### 3.5 适用场景
- 依赖服务响应时间不稳定
- 数据库查询或外部API调用超时
- 需要保护系统免受过长响应时间影响

## 4. 异常比例策略(Error Ratio)

### 4.1 策略原理
基于异常比例的熔断策略。当**异常比例**超过设定阈值，且在统计窗口内的请求数达到最小请求数时触发熔断。

### 4.2 规则配置参数
```java
public class DegradeRule {
    private String resource;          // 资源名称
    private double count;             // 异常比例阈值（0.0-1.0）
    private int timeWindow;           // 熔断时长（单位：s）
    private int grade = RuleConstant.DEGRADE_GRADE_EXCEPTION_RATIO; // 策略类型：异常比例
    private int minRequestAmount;     // 最小触发请求数
    private int statIntervalMs;       // 统计窗口（单位：ms）
}
```

### 4.3 配置示例
```java
DegradeRule rule = new DegradeRule("paymentService")
    .setGrade(RuleConstant.DEGRADE_GRADE_EXCEPTION_RATIO) // 异常比例策略
    .setCount(0.3)                                        // 异常比例阈值：30%
    .setMinRequestAmount(20)                              // 最小请求数：20
    .setStatIntervalMs(60000)                             // 统计窗口：60秒
    .setTimeWindow(10);                                   // 熔断时长：10秒
```

### 4.4 触发条件
1. 统计窗口内请求数 ≥ minRequestAmount
2. 异常比例 ≥ count
   ```
   异常比例 = 异常请求数 / 总请求数
   ```

### 4.5 异常定义
- `BlockException`除外（Sentinel自身阻塞异常）
- 业务异常：`Throwable`或指定异常类型
- 可通过`Tracer.trace(throwable)`手动统计异常

### 4.6 适用场景
- 外部服务返回业务异常
- 参数校验失败率高
- 业务逻辑错误导致的失败

## 5. 异常数策略(Error Count)

### 5.1 策略原理
基于异常数量的熔断策略。当**异常数量**在统计窗口内超过设定阈值时触发熔断。

### 5.2 规则配置参数
```java
public class DegradeRule {
    private String resource;          // 资源名称
    private double count;             // 异常数阈值
    private int timeWindow;           // 熔断时长（单位：s）
    private int grade = RuleConstant.DEGRADE_GRADE_EXCEPTION_COUNT; // 策略类型：异常数
    private int minRequestAmount;     // 最小触发请求数
    private int statIntervalMs;       // 统计窗口（单位：ms）
}
```

### 5.3 配置示例
```java
DegradeRule rule = new DegradeRule("inventoryService")
    .setGrade(RuleConstant.DEGRADE_GRADE_EXCEPTION_COUNT) // 异常数策略
    .setCount(5)                                          // 异常数阈值：5次
    .setMinRequestAmount(1)                               // 最小请求数：1
    .setStatIntervalMs(60000)                             // 统计窗口：60秒
    .setTimeWindow(30);                                   // 熔断时长：30秒
```

### 5.4 触发条件
统计窗口内异常数量 ≥ count

### 5.5 适用场景
- 低流量但重要的服务
- 异常容忍度低的场景
- 需要快速熔断防止错误扩散

## 6. 三种策略对比分析

| 维度 | 慢调用比例 | 异常比例 | 异常数 |
|------|-----------|----------|--------|
| **统计指标** | 响应时间超过阈值的比例 | 异常请求比例 | 异常请求数量 |
| **阈值类型** | 比例(0.0-1.0) | 比例(0.0-1.0) | 整数 |
| **适用场景** | 响应时间敏感型服务 | 异常敏感型服务 | 低流量重要服务 |
| **灵敏度** | 中等 | 高 | 低流量下灵敏度高 |
| **配置复杂度** | 中等（需设置RT阈值和比例） | 简单 | 简单 |

## 7. 最佳实践与配置建议

### 7.1 策略选择指南
1. **慢调用比例策略**：适合对响应时间有严格要求的服务
2. **异常比例策略**：适合异常率敏感的业务场景
3. **异常数策略**：适合低流量但关键的服务

### 7.2 参数调优建议

#### 统计窗口设置
```java
// 根据业务特点设置合适的统计窗口
public class StatIntervalConfig {
    // 高频服务：较短窗口（5-10秒）
    public static final int HIGH_FREQUENCY = 5000;
    
    // 中频服务：中等窗口（10-30秒）
    public static final int MEDIUM_FREQUENCY = 10000;
    
    // 低频服务：较长窗口（30-60秒）
    public static final int LOW_FREQUENCY = 30000;
}
```

#### 最小请求数配置
```java
// 根据服务流量设置
public class MinRequestConfig {
    // 高QPS服务：适当提高最小请求数，避免偶发波动
    private int highQPSMinRequest = 50;
    
    // 低QPS服务：降低最小请求数，保证敏感性
    private int lowQPSMinRequest = 5;
}
```

## 9. 注意事项

### 9.1 避免的陷阱
1. **阈值设置过低**：导致频繁熔断，影响正常业务
2. **统计窗口过短**：对瞬时波动过于敏感
3. **忽略预热期**：服务启动初期可能不稳定
4. **单一策略依赖**：根据业务特点组合使用策略

### 9.2 异常处理建议
```java
@SentinelResource(
    value = "businessMethod",
    blockHandler = "handleBlockException",
    fallback = "fallbackMethod",
    exceptionsToIgnore = {IllegalArgumentException.class}
)
public Response businessMethod(Request request) {
    // 业务逻辑
}

// 熔断降级处理
public Response handleBlockException(Request request, BlockException ex) {
    // 返回降级结果
    return Response.fallback("服务暂时不可用");
}

// 异常降级
public Response fallbackMethod(Request request, Throwable t) {
    // 异常降级逻辑
    return Response.fallback("服务异常，已降级处理");
}
```

## 10. 总结

Sentinel提供的三种熔断策略各有侧重，在实际应用中应根据业务场景灵活选择和配置：

1. **慢调用比例策略**重点关注服务响应时间
2. **异常比例策略**关注服务稳定性质量
3. **异常数策略**适用于低流量关键路径

合理配置熔断策略，结合监控告警，可以有效提升微服务架构的稳定性和韧性，确保系统在异常情况下的自我保护和快速恢复能力。
