# 运行时安全：Falco 系统调用规则检测技术文档

## 1. 概述

### 1.1 运行时安全定义
运行时安全（Runtime Security）是指在应用程序或系统运行过程中，实时检测和防御安全威胁的安全机制。与传统边界安全不同，运行时安全专注于工作负载内部行为监控，尤其适用于云原生和容器化环境。

### 1.2 Falco 简介
Falco 是由 Sysdig 创建的云原生运行时安全项目，现为 CNCF（云原生计算基金会）孵化级项目。它通过监控系统调用（syscall）来检测容器、主机和Kubernetes集群中的异常行为，提供实时的安全威胁检测和警报能力。

## 2. Falco 架构与核心组件

### 2.1 系统架构
```
+-------------------+     +-------------------+     +-------------------+
|   数据源层         |     |   规则引擎层      |     |   输出层          |
|   - Linux内核      |---->|   - 规则文件      |---->|   - 标准输出      |
|   - eBPF探针       |     |   - 规则条件匹配  |     |   - 文件          |
|   - 内核模块       |     |   - 告警触发      |     |   - Webhook       |
+-------------------+     +-------------------+     +-------------------+
```

### 2.2 核心组件

#### 2.2.1 Falco 驱动
- **内核模块**：传统捕获方式，需要编译内核模块
- **eBPF探针**：现代方式，无需内核模块，安全性更高
- **用户态代理**：通过审计日志收集数据（性能较低）

#### 2.2.2 规则引擎
- 基于YAML格式的规则定义
- 支持条件表达式和宏定义
- 可扩展的插件系统

#### 2.2.3 输出系统
- 多种输出格式（JSON、文本等）
- 多种输出目的地（文件、标准输出、网络端点）
- 支持与SIEM系统集成

## 3. Falco 规则系统详解

### 3.1 规则结构
```yaml
- rule: Terminal shell in container
  desc: A shell was spawned by a non-shell program in a container
  condition: >
    container.id != host and
    proc.name = bash and
    evt.type = execve and
    not proc.name in (bash, sh, dash)
  output: >
    Shell spawned in container (user=%user.name container_id=%container.id 
    container_name=%container.name shell=%proc.name parent=%proc.pname 
    cmdline=%proc.cmdline)
  priority: WARNING
  tags: [container, shell, mitre_execution]
```

### 3.2 规则关键字段

| 字段 | 描述 | 必选 |
|------|------|------|
| `rule` | 规则名称 | 是 |
| `desc` | 规则描述 | 是 |
| `condition` | 触发条件表达式 | 是 |
| `output` | 告警输出格式 | 是 |
| `priority` | 告警级别 | 是 |
| `tags` | 分类标签 | 否 |
| `exceptions` | 例外情况 | 否 |

### 3.3 条件表达式语法

#### 3.3.1 基础运算符
- 比较运算符：`=`, `!=`, `<`, `<=`, `>`, `>=`
- 逻辑运算符：`and`, `or`, `not`
- 集合运算符：`in`, `intersects`, `pmatch`

#### 3.3.2 字段类型
```yaml
# 事件字段
evt.time, evt.type, evt.dir, evt.args

# 进程字段
proc.pid, proc.name, proc.exe, proc.cmdline, proc.cwd

# 容器字段
container.id, container.name, container.image

# 文件字段
fd.name, fd.directory, fd.filename

# 用户字段
user.name, user.uid, user.loginuid
```

## 4. 系统调用检测机制

### 4.1 检测原理
Falco 通过以下方式捕获系统调用：

1. **事件捕获**：使用eBPF或内核模块在系统调用入口/出口处插入探针
2. **事件过滤**：基于规则条件过滤无关事件
3. **上下文关联**：将系统调用与进程、容器、用户上下文关联
4. **规则匹配**：应用规则条件进行匹配检测
5. **告警生成**：匹配成功时生成相应告警

### 4.2 关键系统调用监控

#### 4.2.1 进程执行监控
```yaml
- rule: Unexpected binary process
  desc: Detect execution of unexpected binaries
  condition: >
    evt.type = execve and
    proc.name not in (allowed_processes) and
    container.id != host
  priority: CRITICAL
```

#### 4.2.2 文件系统监控
```yaml
- rule: Write below binary dir
  desc: an attempt to write to any file below a set of binary directories
  condition: >
    write_evt and
    fd.directory in (bin_dirs) and
    not proc.name in (shell_binaries, expected_writers)
  priority: ERROR
```

#### 4.2.3 网络活动监控
```yaml
- rule: Outbound connection to miners pool
  desc: Detect outbound connections to common miners pool
  condition: >
    outbound and
    fd.sockfamily = ip and
    evt.type in (connect, sendto) and
    fd.ip in (miner_ips)
  priority: CRITICAL
```

#### 4.2.4 权限提升监控
```yaml
- rule: Set Setuid or Setgid bit
  desc: When the setuid or setgid bits are set via chmod
  condition: >
    evt.type = chmod and
    (evt.arg.mode contains S_ISUID or evt.arg.mode contains S_ISGID)
  priority: WARNING
```

## 5. 规则开发最佳实践

### 5.1 规则编写原则

#### 5.1.1 精确性原则
```yaml
# 不推荐：过于宽泛
condition: proc.name = bash

# 推荐：结合上下文
condition: >
  container.id != host and
  proc.name = bash and
  evt.type = execve and
  not proc.pname in (sshd, bash, sh)
```

#### 5.1.2 性能考虑
```yaml
# 不推荐：频繁触发
condition: evt.type = open

# 推荐：针对性过滤
condition: >
  evt.type = open and
  fd.name contains /etc/passwd and
  container.id != host
```

### 5.2 规则组织与管理

#### 5.2.1 分层规则结构
```
rules/
├── base_rules.yaml      # 基础规则
├── application_rules.yaml # 应用特定规则
├── compliance_rules.yaml # 合规性规则
└── custom_rules.yaml    # 自定义规则
```

#### 5.2.2 使用宏提高可维护性
```yaml
- macro: container
  condition: container.id != host

- macro: sensitive_files
  condition: >
    fd.name startswith /etc/ or
    fd.name startswith /root/ or
    fd.name startswith /home/

- rule: Access sensitive file in container
  desc: Detect access to sensitive files in container
  condition: container and sensitive_files and open_evt
  priority: WARNING
```

## 6. 部署与配置

### 6.1 部署模式

#### 6.1.1 主机级部署
```bash
# 使用eBPF驱动安装
curl -s https://falco.org/repo/falcosecurity-3672BA8F.asc | apt-key add -
echo "deb https://download.falco.org/packages/deb stable main" | tee -a /etc/apt/sources.list.d/falcosecurity.list
apt-get update && apt-get install -y falco
```

#### 6.1.2 Kubernetes部署
```yaml
# Helm Chart部署
helm repo add falcosecurity https://falcosecurity.github.io/charts
helm install falco falcosecurity/falco \
  --set driver.kind=ebpf \
  --set falco.jsonOutput=true
```

### 6.2 配置优化

#### 6.2.1 性能调优
```yaml
# falco.yaml 配置片段
time_format_support: false        # 禁用时间格式支持提高性能
buffered_outputs: false           # 禁用缓冲输出减少延迟
outputs:
  rate: 1                         # 告警速率限制
  max_burst: 10

syscall_event_drops:
  actions:
    - log                         # 记录事件丢弃
    - alert                       # 发出告警
  rate: 0.05                      # 允许5%的事件丢弃率
  max_burst: 1000
```

#### 6.2.2 规则加载优化
```yaml
rules_file:
  - /etc/falco/falco_rules.yaml
  - /etc/falco/falco_rules.local.yaml
  - /etc/falco/rules.d/

rules:
  priority: DEBUG,INFO,NOTICE,WARNING,ERROR,CRITICAL,ALERT,EMERGENCY
  minimal_priority: WARNING        # 只处理WARNING及以上级别
```

## 9. 安全与合规性

### 9.1 安全基准检测
```yaml
# CIS基准检测规则
- rule: CIS - 5.2.1 Ensure permissions on /etc/passwd are configured
  desc: Detect unauthorized modification of /etc/passwd
  condition: >
    open_write and
    fd.name = /etc/passwd and
    not user.name in (root_users)
  priority: ERROR
  tags: [cis, compliance, filesystem]
```

### 9.2 数据保护
```yaml
# 敏感数据访问监控
- rule: Access to sensitive environment variables
  desc: Process accessing sensitive environment variables
  condition: >
    evt.type = execve and
    proc.env contains "AWS_" or
    proc.env contains "KUBERNETES_" or
    proc.env contains "DATABASE_"
  output: >
    Sensitive env var accessed (user=%user.name
    process=%proc.name env=%proc.env)
  priority: WARNING
```

## 10. 故障排查与优化

### 10.1 常见问题排查

#### 10.1.1 性能问题
```bash
# 检查事件速率
falco --stats

# 查看事件丢弃情况
journalctl -u falco | grep "dropped events"

# 性能分析模式
falco -p -o json_output=true
```

#### 10.1.2 规则调试
```bash
# 测试单个规则

# 规则验证
falco --validate rules/my_rules.yaml

# 详细调试输出
falco -v -o log_level=debug
```

### 10.2 性能优化建议

1. **规则优化**
   - 减少通配符使用
   - 优先使用精确匹配
   - 合理设置优先级过滤

2. **配置优化**
   - 调整事件缓冲区大小
   - 启用事件采样（生产环境谨慎使用）
   - 使用eBPF替代内核模块

3. **架构优化**
   - 分布式部署减轻单点压力
   - 按需加载规则文件
   - 使用规则白名单机制

## 11. 结论与展望

Falco 作为云原生运行时安全的标杆工具，通过系统调用监控提供了强大的安全检测能力。其核心优势包括：

1. **深度可见性**：通过内核级监控提供无盲点的安全可见性
2. **高性能**：基于eBPF技术实现低开销的安全监控
3. **灵活性**：可扩展的规则系统和插件架构
4. **云原生友好**：原生支持容器和Kubernetes环境

随着云原生技术的发展，运行时安全将越来越重要。Falco 社区正持续改进以下方向：
- 更智能的异常检测（集成机器学习）
- 更广泛的数据源支持
- 更好的性能优化
- 更紧密的生态系统集成

通过合理配置和持续优化，Falco 能够为现代基础设施提供有效的运行时安全防护，成为云原生安全架构中不可或缺的一环。
