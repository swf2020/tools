# SQL注入防御：预编译参数化查询技术文档

## 1. 概述

### 1.1 文档目的
本文档旨在详细阐述SQL注入攻击的原理、危害，并深入介绍预编译参数化查询作为SQL注入防御的核心技术方案，提供多语言实现示例和最佳实践。

### 1.2 SQL注入风险概述
SQL注入是Web应用中最常见且危害极大的安全漏洞之一，攻击者通过构造恶意SQL语句，实现对数据库的未授权访问、数据窃取、篡改甚至破坏。

## 2. SQL注入攻击原理

### 2.1 攻击示例
```sql
-- 原始查询
SELECT * FROM users WHERE username = '$username' AND password = '$password'

-- 攻击输入
用户名: admin' --
密码: 任意值

-- 生成的恶意SQL
SELECT * FROM users WHERE username = 'admin' --' AND password = '任意值'
```

### 2.2 攻击类型
- **联合查询注入**：使用UNION获取其他表数据
- **布尔盲注**：通过布尔判断推断数据
- **时间盲注**：通过延时响应推断数据
- **报错注入**：利用数据库报错信息泄露数据

## 3. 预编译参数化查询原理

### 3.1 核心概念
预编译参数化查询（Prepared Statements with Parameterized Queries）将SQL代码与数据完全分离，通过以下机制防止注入：

1. **预处理阶段**：SQL语句模板被发送到数据库编译
2. **参数绑定阶段**：用户输入作为参数传递
3. **执行阶段**：数据库使用预编译的语句执行查询

### 3.2 技术优势
- **根本性防御**：参数数据不会被解释为SQL语法
- **性能优化**：同一查询模板可重复使用
- **代码清晰**：提高代码可读性和可维护性

## 4. 各语言实现示例

### 4.1 Java (JDBC)
```java
// ❌ 不安全的方式（字符串拼接）
String sql = "SELECT * FROM users WHERE username = '" + username + "'";
Statement stmt = connection.createStatement();
ResultSet rs = stmt.executeQuery(sql);

// ✅ 安全的方式（预编译参数化查询）
String sql = "SELECT * FROM users WHERE username = ? AND password = ?";
PreparedStatement pstmt = connection.prepareStatement(sql);
pstmt.setString(1, username);
pstmt.setString(2, password);
ResultSet rs = pstmt.executeQuery();
```

### 4.2 Python (Psycopg2 - PostgreSQL)
```python
# ❌ 不安全的方式
cursor.execute("SELECT * FROM users WHERE username = '%s'" % username)

# ✅ 安全的方式
sql = "SELECT * FROM users WHERE username = %s AND password = %s"
cursor.execute(sql, (username, password))
```

### 4.3 C# (.NET)
```csharp
// ❌ 不安全的方式
string sql = "SELECT * FROM users WHERE username = '" + username + "'";
SqlCommand cmd = new SqlCommand(sql, connection);

// ✅ 安全的方式
string sql = "SELECT * FROM users WHERE username = @username AND password = @password";
SqlCommand cmd = new SqlCommand(sql, connection);
cmd.Parameters.AddWithValue("@username", username);
cmd.Parameters.AddWithValue("@password", password);
```

### 4.4 PHP (PDO)
```php
// ❌ 不安全的方式
$sql = "SELECT * FROM users WHERE username = '$username'";
$result = $pdo->query($sql);

// ✅ 安全的方式
$sql = "SELECT * FROM users WHERE username = :username AND password = :password";
$stmt = $pdo->prepare($sql);
$stmt->bindParam(':username', $username);
$stmt->bindParam(':password', $password);
$stmt->execute();
```

### 4.5 Node.js (mysql2)
```javascript
// ❌ 不安全的方式
connection.query(`SELECT * FROM users WHERE username = '${username}'`);

// ✅ 安全的方式
const sql = 'SELECT * FROM users WHERE username = ? AND password = ?';
connection.execute(sql, [username, password]);
```

## 5. 存储过程中的参数化查询

### 5.1 存储过程调用
```sql
-- 创建存储过程
CREATE PROCEDURE GetUserByCredentials
    @Username NVARCHAR(50),
    @Password NVARCHAR(50)
AS
BEGIN
    SELECT * FROM Users 
    WHERE Username = @Username AND Password = @Password
END
```

```csharp
// C# 调用存储过程
using (SqlCommand cmd = new SqlCommand("GetUserByCredentials", connection))
{
    cmd.CommandType = CommandType.StoredProcedure;
    cmd.Parameters.AddWithValue("@Username", username);
    cmd.Parameters.AddWithValue("@Password", password);
}
```

## 6. ORM框架中的参数化查询

### 6.1 Entity Framework (C#)
```csharp
// LINQ查询自动参数化
var user = context.Users
    .Where(u => u.Username == username && u.Password == password)
    .FirstOrDefault();

// 原始SQL查询手动参数化
var user = context.Users
    .FromSqlRaw("SELECT * FROM Users WHERE Username = {0} AND Password = {1}", 
                username, password)
    .FirstOrDefault();
```

### 6.2 Hibernate (Java)

## 7. 特殊情况处理

### 7.1 动态表名/列名
```java
// ❌ 错误：表名不能参数化
String sql = "SELECT * FROM ? WHERE id = ?";

// ✅ 正确：使用白名单验证
Set<String> allowedTables = new HashSet<>(Arrays.asList("users", "products"));
if (!allowedTables.contains(tableName)) {
    throw new SecurityException("Invalid table name");
}
String sql = "SELECT * FROM " + tableName + " WHERE id = ?";
PreparedStatement pstmt = connection.prepareStatement(sql);
pstmt.setInt(1, id);
```

### 7.2 IN语句参数化
```java
// 构建动态参数占位符
List<Integer> ids = Arrays.asList(1, 2, 3, 4);
String placeholders = String.join(",", 
    Collections.nCopies(ids.size(), "?"));

String sql = "SELECT * FROM products WHERE id IN (" + placeholders + ")";
PreparedStatement pstmt = connection.prepareStatement(sql);

for (int i = 0; i < ids.size(); i++) {
    pstmt.setInt(i + 1, ids.get(i));
}
```

## 8. 最佳实践与注意事项

### 8.1 安全编码准则
1. **始终使用参数化查询**：即使是可信来源的数据
2. **最小权限原则**：数据库用户应只有必要权限
3. **输入验证**：在参数化前进行输入格式验证
4. **错误处理**：避免详细错误信息泄露给用户
5. **日志记录**：记录所有数据库操作便于审计

### 8.2 性能考虑
- **连接池管理**：合理配置连接池参数
- **批量操作优化**：使用批量预编译语句
- **查询计划重用**：充分利用预编译的性能优势

### 8.3 测试与验证
```java
// 安全测试示例
@Test
public void testSqlInjectionPrevention() {
    // 尝试注入攻击
    String maliciousInput = "admin' OR '1'='1";
    
    // 验证参数化查询是否安全
    String sql = "SELECT COUNT(*) FROM users WHERE username = ?";
    PreparedStatement pstmt = connection.prepareStatement(sql);
    pstmt.setString(1, maliciousInput);
    
    // 应该查询用户名为 "admin' OR '1'='1" 的记录，而不是所有记录
    ResultSet rs = pstmt.executeQuery();
    rs.next();
    int count = rs.getInt(1);
    
    assertEquals(0, count); // 应该返回0，除非真的有这个奇怪用户名的用户
}
```

## 10. 总结

预编译参数化查询是防御SQL注入攻击最有效、最根本的技术方案。通过将SQL代码与数据分离，它能彻底防止攻击者将恶意数据解释为SQL指令。尽管需要开发者改变传统的字符串拼接习惯，但带来的安全收益和代码质量提升是显著的。

建议所有涉及数据库操作的开发团队都将参数化查询作为强制性编码规范，并结合其他安全措施构建多层次的防御体系，确保应用系统的数据安全。
