## 平台特征

- 传统服务端渲染页面，部分内容可通过 WebFetch 直接抓取
- 搜索结果页 URL 结构稳定，参数含义明确
- 有分页限制（通常显示前 5 页），翻页频率过高可能触发验证码
- 反爬策略中等：IP 频率限制 + 偶尔验证码
- 无需登录即可浏览搜索列表和职位详情

## 有效模式

### 搜索 URL 结构

```
https://search.51job.com/list/{cityCode},000000,0000,00,9,99,{keyword},2,{page}.html
```

参数说明：
- `{cityCode}` — 6 位城市编码（如 040000=深圳，010000=北京）
- `000000` — 职能类别编码（000000=不限）
- `0000` — 行业类别编码（0000=不限）
- `00` — 薪资范围编码（00=不限）
- `9` — 公司性质（9=不限）
- `99` — 工作经验（99=不限）
- `{keyword}` — 搜索关键词（URL 编码）
- `2` — 搜索模式（2=关键词搜索）
- `{page}` — 页码（从 1 开始）

### CDP 浏览器方式（推荐）

- 打开搜索 URL，等待页面加载完成后通过 eval 提取 DOM
- 职位列表容器：`#resultList` 下的 `.el` 行（class="el" 的 div/tr）
- 每条职位的数据在对应 `.el` 的子元素中：
  - 岗位名：`.t1 span a` 的 text
  - 公司名：`.t2 a` 的 text
  - 工作地点：`.t3` 的 text
  - 薪资：`.t4` 的 text
  - 发布日期：`.t5` 的 text
- 职位详情链接：`.t1 span a` 的 href

### WebFetch 方式（有限）

- WebFetch 可获取搜索结果列表的 HTML
- 提取各字段后直接组装 JSON，无需进入详情页
- 注意编码：51job 页面通常为 GBK 编码

## 城市编码参考

城市编码数据在官方 JS 文件中定义：
```
https://js.51jobcdn.com/in/js/2016/layer/area_array_c.js
```

格式：`var area = {code: "城市名", ...}` (GBK 编码)

常用城市编码（2026-05-16 验证）：

## 已知陷阱

- 翻页频率过高会触发验证码，建议每页间隔 3-5 秒
- 部分职位详情页为动态加载（AJAX），需要 CDP 浏览器获取完整内容
- 搜索结果列表页的 HTML 结构可能因 A/B 测试而变化，建议使用 CDP 浏览器以确保一致性
- 页面编码为 GBK，WebFetch 方式需注意字符编码转换
